# Layer 1 — Core Context (stable spine)

> Load this in **every** session. It rarely changes. Bulky facts live in `reference/`.

## 1. What this is

**DancePlatform** is a dance learning/tracking app. Users browse a catalog of dances,
watch embedded clips (YouTube / TikTok / Instagram), mark dances **Favorite / In Progress /
Learned**, rate them 1–5, log practice sessions, and keep a public or private profile.
Admins curate the catalog (dances, videos, styles, instructors, bulk import).

## 3. Architecture

```
Browser (Angular SPA)
  │  HTTP + JWT (Bearer) ; apiUrl from src/environments/*
  ▼
ASP.NET Core API  ── Controllers → Service interfaces (IXxxService) → Service impls → EF Core → PostgreSQL
                     RequireAdmin / [Authorize] filters gate access
                     SeedData runs on startup (only if Dances table empty)
```

**API layering (strict — follow it for every new feature):**
`Controller` → `IXxxService` (interface) → `XxxService` (impl, holds `AppDbContext`) → EF Core.
Controllers never touch `AppDbContext` directly; they call the injected service interface.
DTOs (in `DTOs/`) cross the boundary — **never** return EF entity classes from a controller.

**Frontend layering:**
`Component (standalone, signals)` → `core/services/*.service.ts` (HttpClient) → API.
Models in `models/*.model.ts`. Routes are lazy-loaded standalone components.

## 5. Conventions

- **Every entity has a `DateAdded` (`DateTime`, UTC) timestamp.** Add it to new models.
- **Slugs:** dances have a unique `Slug` generated by `Services/SlugGenerator.Slugify`
  (lowercase, diacritics stripped, non-alphanumerics → `-`). Detail route is by slug.
- **DTO in, DTO out.** Requests are `CreateXxxRequest` / `UpdateXxxRequest`; responses are
  `XxxDto`. Keep them in `DTOs/<Area>/`.
- **Service registration:** every service is `AddScoped<IXxx, Xxx>()` in `Program.cs`.
- **Many-to-many** join entities are explicit classes (`DanceStyle`, `DanceMusicalStyle`,
  `DanceInstructor`, `UserMyStyle`, and the user↔dance status joins) with composite keys
  configured in `AppDbContext.OnModelCreating`.
- **Angular state** is held in **signals** (`signal`, `computed`), not BehaviorSubjects, in
  the newer code (e.g. `auth.service.ts`). Match the surrounding component's style.
- **No emojis** in product UI/text (per project design rule: "warm and energetic").
- **Comments:** only where logic is non-obvious. Don't narrate.

## 6. Auth model (read before touching anything secured)

- Login/register → API returns `{ token, userId, username }`. FE stores token under
  `localStorage['dp_token']` and `{userId,username}` under the user key.
- **The JWT carries `NameIdentifier` (user id), `Name` (username), and a signed `isAdmin`
  claim** (`AuthService.GenerateToken` stamps it from `User.IsAdmin` at login/register).
- **Admin is read from that signed claim — no per-request DB lookup, no `/role/me` endpoint.**
  `RequireAdminAttribute` checks `isAdmin == "true"`; the frontend reads it client-side via
  `jwtIsAdmin()` (`core/utils/jwt.utils.ts`), used by `RoleService.loadFromToken()` and
  `adminGuard`. Trusting it is safe because the HMAC signature is verified on every request.
- **Trade-off:** the claim is fixed for the token's 30-day life, so a new grant/revoke only
  takes effect on the user's next login. (Set `Users.IsAdmin` in the DB, then re-login.)
- `[Authorize]` = any logged-in user. `[RequireAdmin]` = signed `isAdmin` claim is `"true"`.
- Current-user id inside controllers comes from the `NameIdentifier` claim
  (`CurrentUserId`), not from request input.

## 7. Local run / deploy (quick reference)

- API: `dotnet run` in `DancePlatform.API/` (listens on `http://localhost:5000`; Swagger in dev).
  On boot it auto-applies migrations and seeds if empty.
- UI: `npm start` (or `ng serve`) in `dance-platform-ui/` → `http://localhost:4200`.
  Dev API base = `http://localhost:5000/api`; prod = `https://dance-api.takelord.com/api`.
- Migrations are added by hand: `dotnet ef migrations add <Name> --no-build` then they apply
  on next API start (`db.Database.MigrateAsync()` in `Program.cs`).
- Production target: Raspberry Pi — PostgreSQL service, API as systemd service, Angular
  served via Apache2 reverse proxy. Live at `dance.takelord.com` / API `dance-api.takelord.com`.

## 8. ⛔ DO NOT (hard rules — violating these breaks the system or its conventions)

1. **Do not trust a client-sent admin flag (request body/header/query).** Admin comes
   only from the **signed `isAdmin` JWT claim** verified by `RequireAdminAttribute`. The
   signature makes the claim tamper-proof; an unsigned/client-supplied flag must never gate
   anything. When granting admin, remember the claim is fixed until the user's next login.
2. **Do not return EF entity classes from controllers.** Map to a DTO. Returning entities
   leaks navigation properties / causes serialization cycles.
3. **Do not bypass the service layer.** Controllers call `IXxxService`, never `AppDbContext`.
4. **Do not invent endpoints, columns, DTO fields, or query params.** If it's not in
   `reference/api-contracts.md` or `reference/db-schema.md`, it doesn't exist — ask/add it
   properly (model → AppDbContext → DTO → service → controller → migration).
5. **Do not change the DB without an EF migration.** No raw schema edits in code paths;
   add a migration. (One-off prod *data* cleanups are done by hand on the Pi with a backup
   first — see known-issues #5 — but **schema** always goes through migrations.)
6. **Do not derive "today"/dates from UTC for user-facing day logic.** Use the user's
   **local** date. This caused practice-streak bugs (known-issues #1/#2). For `DateOnly`
   strings on the FE, parse as local midnight (`dateStr + 'T00:00:00'`), never `new Date(dateStr)`.
7. **Do not assume profiles are public.** `ProfileVisibility` defaults to **Private**.
   `GET /api/users/{username}` returns a profile **only if Public**; respect that.
8. **Do not allow duplicate dances silently in new code paths.** The API currently accepts
   duplicate names and auto-suffixes the slug (`reebok-2`…); this caused data-quality issues.
   New create flows should warn on name collision.
9. **Do not upgrade Angular/.NET major versions or swap core libs** (Npgsql, BCrypt, JWT,
   signals→NgRx, etc.) unless explicitly asked. The "Angular v22" note in `copilot.txt` is wrong.
10. **Do not add emojis** to product copy/UI.
11. **Do not store secrets in the repo.** JWT key / connection string come from configuration
    (`Jwt:Key`, `ConnectionStrings:Default`), not source.
12. **Do not remove `DateAdded`** from entities or forget it on new ones.
